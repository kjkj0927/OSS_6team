## 🛠️ 작업 방식 (Workflow)
본 프로젝트는 다음과 같은 Git/GitHub 협업 방식을 준수하여 진행합니다.

1. **Branch 생성:** 각 팀원은 `main` 브랜치에서 분기하여 개인 브랜치(예: `feature/북문` 또는 `feature/홍길동`)를 생성합니다.
2. **파일 작성:** 할당받은 출입구의 맛집 정보를 개별 `.md` 파일로 작성하고 커밋합니다.
3. **Merge:** 작성이 완료되면 `main` 브랜치로 병합(Merge)합니다.
4. **구조화:** 모든 파일이 모이면 조장이 `README.md`에 각 파일을 하이퍼링크로 연결하여 최종 완성합니다.

초기 단계에서는 각 팀원이 개별적으로 경북대학교 정문, 북문, 쪽문, 텍문 주변 맛집을 조사한 뒤 기준에 맞는 식당을 선정하고 이를 md 파일 형태로 정리하였다. 이후 팀원 간 서로의 md 파일을 공유하고 내용을 검토하는 과정을 거쳤으며, 식당 선정의 적절성, 정보의 정확성, 그리고 전체적인 구성의 통일성을 중심으로 피드백을 주고받았다.
